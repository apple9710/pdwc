# 커밋 규칙

| 커밋 타입 | 설명                                            |
| --------- | ----------------------------------------------- |
| feat      | 새로운 기능 추가                                |
| fix       | 버그 수정                                       |
| modify    | 기존 기능 수정                                  |
| style     | 코드 포맷팅, 세미콜론 누락 등 (코드 변경 없음)  |
| refactor  | 코드 리팩토링                                   |
| docs      | 문서 수정                                       |
| remove    | 파일/코드 삭제                                  |
| build     | 빌드 시스템, 외부 종속성 수정                   |
| chore     | 기타 변경사항 (빌드 스크립트, 패키지 매니저 등) |
| test      | 테스트 코드 추가/수정                           |
| perf      | 성능 개선                                       |
| ci        | CI/CD 설정 파일 수정                            |
| revert    | 이전 커밋 되돌리기                              |
| typo      | 오타 수정                                       |
| rename    | 파일/폴더명 변경                                |
| comment   | 주석 추가/수정                                  |

...업데이트중
