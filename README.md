## VSCODE (다른거 쓰면 알려달라 해주세여)

### SSH 를 사용하지 않으실 경우! (첫 실행)

- VSCODE에서 터미널 열구

```bash
git clone https://github.com/apple9710/pdwc.git
```

- 하면 프로젝트 복사 완료

### 이후 업데이트

- 터미널에서

```bash
  ## [main] 이건 브랜치명
  ## 브랜치가 다를 경우 다른거 입력
  ## 예시) git pull origin gived0
  git pull origin main
```

### 커밋을 알아보자

- 귀찮은데 다 업데이트 하기

```bash
  ## 모든 파일 스테이징 - 올리기 전 대기 상태
  git add .
```

```bash
  ## 커밋 메세지는 아래 커밋 규칙을 지키면서 작성 하는 연습을 합시다. - 저도 잘 안지켜요
  git commit -m '커밋메세지'
```

```bash
  ## 보통은 메인 보다는 dev 브랜치를 만들어서 따로 관리 합니다.
  git push origin main
```

> **이거까지 하면 업데이트 완료!**

# 작성 규칙

- 각자 자기 폴더에서만 수정하기!
- 다른 사람 폴더 수정 하기 금지!

# 커밋 규칙

| 커밋 타입 | 설명                                            |
| --------- | ----------------------------------------------- |
| feat      | 새로운 기능 추가                                |
| fix       | 버그 수정                                       |
| modify    | 기존 기능 수정                                  |
| style     | 코드 포맷팅, 세미콜론 누락 등 (코드 변경 없음)  |
| refactor  | 코드 리팩토링                                   |
| docs      | 문서 수정                                       |
| remove    | 파일/코드 삭제                                  |
| build     | 빌드 시스템, 외부 종속성 수정                   |
| chore     | 기타 변경사항 (빌드 스크립트, 패키지 매니저 등) |
| test      | 테스트 코드 추가/수정                           |
| perf      | 성능 개선                                       |
| ci        | CI/CD 설정 파일 수정                            |
| revert    | 이전 커밋 되돌리기                              |
| typo      | 오타 수정                                       |
| rename    | 파일/폴더명 변경                                |
| comment   | 주석 추가/수정                                  |

...업데이트중
